API Documentation

Complete REST API reference for the Zorvyn Finance Dashboard — authentication, financial records, analytics, and user management.

Base URL http://localhost:5000/api/v1

🔑 Authentication

All protected endpoints require a JWT Bearer token. Obtain a token via the Login endpoint, then include it in every request header:

Authorization: Bearer <your_jwt_token>
ℹ️ Tokens expire after 30 days. Re-authenticate to obtain a fresh token.

🛡️ Roles & Permissions

Access is scoped per-company. Users only see data belonging to their organization.

Request Body
FieldTypeDescription

companyName*stringMust match registration

{

  "token": "eyJhbGciOiJIUzI1NiIs...",

⚠️ Deactivated users receive a 403 error.

Financial Records

Company-scoped CRUD with filtering, sorting, and pagination. Deletion is soft-delete.

GET /records List with filters & pagination ▼
Access
AdminAnalyst
Query Parameters
ParamTypeDescription
typestringincome | expense
categorystringRegex search on category
dateFromdateStart date (ISO)
dateTodateEnd date (ISO)
sortstringField to sort, e.g. -date (default)
pageintPage number (default: 1)
limitintResults per page (1–50, default: 10)
Response — 200
{
  "status": "success", "results": 10,
  "data": {
    "records": [ { "amount": 5000, "type": "income", ... } ],
    "pagination": { "page": 1, "limit": 10, "total": 42, "totalPages": 5 }
  }
}
GET /records/:id Get single record ▼
Access
AdminAnalyst
URL Parameters
ParamTypeDescription

POST /records Create new record ▼
Access
Admin
Request Body
FieldTypeDescription
amount*numberTransaction amount
type*stringincome | expense
category*stringe.g. Salary, Marketing
datedateISO date (default: now)
notesstringMax 500 chars
ℹ️ createdBy and companyName are set automatically from the JWT.
PATCH /records/:id Update record ▼
Access
Admin

Allowed fields: amount, type, category, date, notes

DELETE /records/:id Soft-delete record ▼
Access
Admin
⚠️ This is a soft delete — the record is flagged as deleted but not removed from the database.

User Management

Admin-only. All operations are scoped to the admin's company.

GET /users List company users ▼
Access
Admin
PATCH /users/:id/role Change user role ▼
Access
Admin
Request Body
FieldTypeDescription
role*stringViewer | Analyst | Admin
⚠️ Admins cannot change their own role.
PATCH /users/:id/status Activate / Deactivate user ▼
Access
Admin

Toggles isActive. Deactivated users cannot log in.

DELETE /users/:id Permanently delete user ▼
Access
Admin
⚠️ This is a hard delete. The user record is permanently removed. Admins cannot delete themselves.
